# linkseek 生产部署指南

> 目标服务器：火山云 `115.190.219.215`（Ubuntu 24.04）
> 域名：`linkseek.honlnk.com`
> 前提：服务器已装 Docker + Compose，共享网关 `honlnk-gateway`（Nginx 容器）在跑并占用 `0.0.0.0:80/443`

---

## 架构概览（双层网关模式）

```
公网:443
  │
  ▼
honlnk-gateway (共享 Nginx，全机唯一 80/443 入口)
  │  外层 HTTPS 终止（证书 /etc/nginx/ssl-linkseek）
  │  proxy_pass https://linkseek-gateway:443
  ▼
linkseek-gateway (项目专属 Nginx，容器内 HTTPS，不映射宿主机端口)
  │  自管路由：SSE 长连接 / SPA 回退 / 安全头 / body 大小
  ▼
linkseek-app:7300
  ├── linkseek-mysql(容器:3306)
  ├── linkseek-searxng(容器:8080，经 mihomo 代理访问搜索引擎)
  └── linkseek-browser-fetch(容器:3000，Playwright 渲染)
```

**为什么要双层网关**：服务器单 IP 上 `80/443` 只能被一个容器绑定（已被 `honlnk-gateway` 占用）。因此 linkseek 不能自己绑这两个端口，而是拥有一个**项目专属的 `linkseek-gateway` 容器**——它跨两个网络（`linkseek-net` 内部 + `honlnk-gateway-net` 对外），`honlnk-gateway` 只做按域名分流转发过来，所有 linkseek 特有的路由配置收在项目自管的 `linkseek-gateway` 里。这与 piclist/obsidian 那种"直接往共享网关注塞 conf"的轻量做法区分开——linkseek 作为独立项目，网关配置自洽、随仓库版本管理。

## 服务器部署目录

服务器上只保留**部署必需文件**（不含源码），与 dai-dai 项目的部署结构对齐：

```
/home/honlnk/linkseek/
├── docker-compose.prod.yml      # 服务编排
├── .env                         # MYSQL_ROOT_PASSWORD / MYSQL_USER / MYSQL_PASSWORD（不进 git）
├── env/
│   └── app.env                  # app 运行时配置（不进 git）
├── nginx/
│   └── linkseek-gateway.conf    # 项目专属网关配置
├── searxng/
│   └── settings.yml             # SearXNG 配置（bind mount 进容器）
└── ssl/                         # 证书（renew-hook.sh 续期时自动更新）
    ├── fullchain.pem
    └── privkey.pem
```

> 镜像 `honlnk/linkseek:latest` 和 `honlnk/linkseek-browser-fetch:latest` 从 Docker Hub 拉取，服务器上无需源码。

## 全新部署步骤

### 第 1 步：配置 DNS

在阿里云 DNS 控制台，为 `honlnk.com` 添加 A 记录：

| 记录类型 | 主机记录 | 记录值 |
|---------|---------|--------|
| A | linkseek | 115.190.219.215 |

验证（DNS 生效可能需几分钟）：
```bash
dig +short linkseek.honlnk.com
# 应返回 115.190.219.215
```

### 第 2 步：申请 SSL 证书

在服务器上执行（certbot 已装）。证书续期由 `renew-hook.sh` 自动处理（见第 8 步）。

```bash
# 先临时停共享网关释放 80 端口
sudo docker stop honlnk-gateway
sudo certbot certonly --standalone -d linkseek.honlnk.com
sudo docker start honlnk-gateway

# 拷贝证书到 linkseek 目录
mkdir -p /home/honlnk/linkseek/ssl
sudo cp /etc/letsencrypt/live/linkseek.honlnk.com/fullchain.pem /home/honlnk/linkseek/ssl/
sudo cp /etc/letsencrypt/live/linkseek.honlnk.com/privkey.pem /home/honlnk/linkseek/ssl/
sudo chown -R honlnk:honlnk /home/honlnk/linkseek/ssl
```

> certbot 自带 systemd timer 自动续期，配合第 8 步的 renew-hook 自动刷新证书。

### 第 3 步：准备部署目录

从 GitHub 拉取仓库后，只保留部署文件：
```bash
cd /home/honlnk
git clone https://github.com/honlnk/linkseek.git
cd linkseek

# 部署只需要这些文件（其余 src/web/Dockerfile 等是本地构建用的）
# docker-compose.prod.yml / deploy/nginx/ / searxng/ / .env.production.example
```

### 第 4 步：配置生产环境变量

```bash
# MySQL 凭据（docker compose 从同目录 .env 读取 ${MYSQL_*}）
cp .env.production.example .env.production
```

编辑 `.env.production`（或拆成 `.env` + `env/app.env`），替换所有 `__XXX__` 占位符：
```bash
openssl rand -hex 32    # MYSQL_ROOT_PASSWORD
openssl rand -hex 16    # MYSQL_PASSWORD（记下来，DATABASE_URL 里要一致）
openssl rand -hex 32    # SESSION_SECRET
```

**关键**：`DATABASE_URL` 里的密码必须和 `MYSQL_PASSWORD` 一致。

参考目录结构把 MySQL 凭据放 `.env`、应用配置放 `env/app.env`（两者均已在 `.gitignore`）。

### 第 5 步：放置项目专属网关配置

```bash
mkdir -p /home/honlnk/linkseek/nginx
cp deploy/nginx/linkseek-gateway.conf /home/honlnk/linkseek/nginx/
```

### 第 6 步：确认共享网关网络

```bash
# honlnk-gateway-net 必须存在，linkseek-gateway 通过它接入对外网络
docker network ls | grep honlnk-gateway-net
```

### 第 7 步：启动容器

```bash
cd /home/honlnk/linkseek
docker compose -f docker-compose.prod.yml up -d

# 查看启动状态（5 个容器都应是 Up / healthy）
docker compose -f docker-compose.prod.yml ps
```

查看 app 日志确认迁移成功：
```bash
docker logs linkseek-app
# 应看到 "All migrations have been successfully applied"
# "✅ 管理员账号已就绪"
# "linkseek 服务已启动"
```

> 迁移和管理员初始化在容器启动时自动执行（幂等），无需手动跑 seed。

### 第 8 步：配置 honlnk-gateway 分流

把 linkseek 的分流配置塞进共享网关（只做 HTTPS 终止 + 转发到 linkseek-gateway）：
```bash
sudo cp deploy/nginx/linkseek.honlnk.com.conf /home/honlnk/honlnk-gateway/conf.d/

# 确认 gateway 已挂载 linkseek 证书目录
docker inspect honlnk-gateway | grep ssl-linkseek
# 应看到 /home/honlnk/linkseek/ssl -> /etc/nginx/ssl-linkseek

# 测试 + 重载
docker exec honlnk-gateway nginx -t
docker exec honlnk-gateway nginx -s reload
```

### 第 9 步：配置证书续期钩子

在 `/home/honlnk/honlnk-gateway/renew-hook.sh` 的 reload 段，新增一行 reload linkseek-gateway（外层 + 内层网关都要刷新证书）：
```bash
docker exec linkseek-gateway nginx -s reload 2>/dev/null
```

> renew-hook.sh 里 linkseek 证书的拷贝逻辑已存在（拷到 `/home/honlnk/linkseek/ssl/`），只需确保末尾同时 reload 两个网关。

### 第 10 步：验证

```bash
# 健康检查
curl https://linkseek.honlnk.com/health
# {"status":"ok"}

# MCP 握手（先在后台建 Key）
curl https://linkseek.honlnk.com/mcp \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -d '{"jsonrpc":"2.0","method":"initialize","params":{"protocolVersion":"2025-03-26","capabilities":{},"clientInfo":{"name":"test","version":"1.0"}},"id":1}'
```

浏览器打开 `https://linkseek.honlnk.com`，用 `ADMIN_PASSWORD` 登录后台。

## 日常维护

### 更新版本

镜像发布在 Docker Hub（`honlnk/linkseek:latest`）。更新流程：
```bash
cd /home/honlnk/linkseek
docker compose -f docker-compose.prod.yml pull      # 拉取最新镜像
docker compose -f docker-compose.prod.yml up -d      # 用新镜像重启（迁移自动执行）
```

> 如果 `docker-compose.prod.yml` 或 `nginx/`、`searxng/` 有改动，先 `git pull` 拉取配置，再 pull 镜像。

### 查看日志

```bash
# app 实时日志
docker compose -f docker-compose.prod.yml logs -f app

# 网关访问日志
docker logs --tail 100 linkseek-gateway
```

### 数据库备份

```bash
# 手动备份
docker exec linkseek-mysql mysqldump -u linkseek -p$(grep MYSQL_PASSWORD .env | cut -d= -f2) linkseek > backup_$(date +%Y%m%d).sql

# 恢复
docker exec -i linkseek-mysql mysql -u linkseek -pPASSWORD linkseek < backup_20260713.sql
```

### 重启服务

```bash
# 仅重启 app
docker compose -f docker-compose.prod.yml restart app

# 仅 reload 网关配置（改 nginx 配置后）
docker exec linkseek-gateway nginx -t && docker exec linkseek-gateway nginx -s reload
```

### 停止/启动全部

```bash
docker compose -f docker-compose.prod.yml down      # 停止（保留数据）
docker compose -f docker-compose.prod.yml up -d      # 启动
docker compose -f docker-compose.prod.yml down -v    # ⚠️ 停止并删除数据（危险）
```

## 故障排查

| 症状 | 排查 |
|------|------|
| `curl` 连接超时 | DNS 未生效 / 安全组未放行 443 |
| 502 Bad Gateway（外层） | `linkseek-gateway` 容器未启动，或不在 `honlnk-gateway-net` 网络 |
| 502 Bad Gateway（内层） | `linkseek-app` 未就绪；查 `docker logs linkseek-gateway` 看是哪一层 |
| 502 + app 日志 ECONNREFUSED | MySQL 未就绪，等一会或检查 mysql 容器健康状态 |
| 证书错误 | `sudo certbot certificates` 检查；确认 renew-hook 里两个 gateway 都 reload 了 |
| 登录失败 "管理员账号未初始化" | 检查 `env/app.env` 的 ADMIN_PASSWORD，容器启动日志看初始化段 |
| MCP 调用 401 | Key 不存在或已禁用；在后台检查 |
| 改了 nginx 配置不生效 | `docker exec linkseek-gateway nginx -t` 看语法，再 `-s reload` |
