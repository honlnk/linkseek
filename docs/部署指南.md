# linkseek 生产部署指南

> 目标服务器：火山云 `115.190.219.215`（Ubuntu 24.04）
> 域名：`linkseek.honlnk.com`
> 前提：服务器已装 Docker + Compose，`honlnk-gateway`（Nginx 容器）在跑

---

## 架构概览

```
linkseek.honlnk.com → honlnk-gateway(Nginx:443) → linkseek-app(容器:3000)
                                                    ├── linkseek-mysql(容器:3306)
                                                    └── linkseek-searxng(容器:8080)
```

三个 linkseek 容器在独立网络 `linkseek-net`，与呆呆项目完全隔离。
`linkseek-app` 同时接入 `honlnk-gateway-net`，让 Nginx 能反代到它。

---

## 部署步骤

### 第 1 步：配置 DNS

在阿里云 DNS 控制台，为 `honlnk.com` 添加 A 记录：

| 记录类型 | 主机记录 | 记录值 |
|---------|---------|--------|
| A | linkseek | 115.190.219.215 |

验证（DNS 生效可能需几分钟）：
```bash
dig +short linkseek.honlnk.com
# 应返回 115.190.219.215
```

### 第 2 步：申请 SSL 证书

在服务器上执行（certbot 已装）：

```bash
# 先确保 80 端口可用（临时停一下 gateway 或用 standalone）
# 方式 A：standalone（推荐，需临时占用 80 端口）
sudo docker stop honlnk-gateway
sudo certbot certonly --standalone -d linkseek.honlnk.com
sudo docker start honlnk-gateway

# 验证证书
sudo ls /etc/letsencrypt/live/linkseek.honlnk.com/
# 应有 fullchain.pem  privkey.pem
```

> certbot 自带 systemd timer 自动续期，无需额外配置。

### 第 3 步：拉取配置文件

镜像已发布到 Docker Hub（`honlnk/linkseek:latest`，双架构 amd64/arm64），服务器上**无需构建**。
只需要 docker-compose 配置 + searxng 配置 + 环境变量。

```bash
cd /home/honlnk
git clone https://github.com/honlnk/linkseek.git
cd linkseek
```

> 如果不想 clone 整个仓库，最少只需要这几个文件：
> `docker-compose.prod.yml`、`searxng/settings.yml`、`.env.production.example`、`deploy/nginx/linkseek.honlnk.com.conf`

### 第 4 步：配置生产环境变量

```bash
cp .env.production.example .env.production
```

编辑 `.env.production`，替换所有 `__XXX__` 占位符：

```bash
# 生成随机密码（在服务器上跑）
openssl rand -hex 32    # 用作 MYSQL_ROOT_PASSWORD
openssl rand -hex 32    # 用作 SESSION_SECRET
openssl rand -hex 16    # 用作 MYSQL_PASSWORD（记下来，两处要一致）

# 设置 ADMIN_PASSWORD（管理后台登录密码，自己定一个强密码）
```

**关键**：`DATABASE_URL` 里的密码必须和 `MYSQL_PASSWORD` 一致。

### 第 5 步：启动容器（拉取镜像）

```bash
# 确认 honlnk-gateway-net 网络存在
docker network ls | grep honlnk-gateway-net

# 拉取镜像 + 启动（首次约 1-2 分钟，主要是下载镜像）
docker compose -f docker-compose.prod.yml up -d

# 查看启动状态
docker compose -f docker-compose.prod.yml ps
# 三个容器都应是 Up/healthy
```

查看 app 日志确认迁移成功：
```bash
docker logs linkseek-app
# 应看到 "All migrations have been successfully applied"
# "✅ 管理员账号已就绪"
# 和 "linkseek 服务已启动"
```

> 迁移和管理员初始化在容器启动时自动执行（幂等），无需手动跑 seed。

### 第 6 步：配置 Nginx 反代

```bash
# 复制配置到 gateway 的配置目录
sudo cp deploy/nginx/linkseek.honlnk.com.conf /home/honlnk/honlnk-gateway/conf.d/

# 挂载 Let's Encrypt 证书到 gateway 容器
# 如果 gateway 已挂载 /etc/letsencrypt，跳过；否则需修改 gateway 的 compose 加挂载
# 确认：docker inspect honlnk-gateway | grep letsencrypt

# 测试配置语法
docker exec honlnk-gateway nginx -t

# 重载 Nginx（不停服）
docker exec honlnk-gateway nginx -s reload
```

### 第 7 步：验证

```bash
# 健康检查
curl https://linkseek.honlnk.com/health
# {"status":"ok"}

# MCP 握手（先在后台建 Key）
curl https://linkseek.honlnk.com/mcp \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -d '{"jsonrpc":"2.0","method":"initialize","params":{"protocolVersion":"2025-03-26","capabilities":{},"clientInfo":{"name":"test","version":"1.0"}},"id":1}'
```

浏览器打开 `https://linkseek.honlnk.com`，用 `ADMIN_PASSWORD` 登录后台。

## 日常维护

### 更新版本

镜像发布在 Docker Hub（`honlnk/linkseek:latest`）。更新流程：

```bash
cd /home/honlnk/linkseek
docker compose -f docker-compose.prod.yml pull      # 拉取最新镜像
docker compose -f docker-compose.prod.yml up -d      # 用新镜像重启（迁移自动执行）
```

> 如果 `docker-compose.prod.yml` 或 `searxng/settings.yml` 有改动，先 `git pull` 拉取配置，再 pull 镜像。

### 查看日志

```bash
# 实时日志
docker compose -f docker-compose.prod.yml logs -f app

# 最近 100 行
docker logs --tail 100 linkseek-app
```

### 数据库备份

```bash
# 手动备份
docker exec linkseek-mysql mysqldump -u linkseek -p$(grep MYSQL_PASSWORD .env.production | cut -d= -f2) linkseek > backup_$(date +%Y%m%d).sql

# 恢复
docker exec -i linkseek-mysql mysql -u linkseek -pPASSWORD linkseek < backup_20260713.sql
```

### 重启服务

```bash
docker compose -f docker-compose.prod.yml restart app
```

### 停止/启动全部

```bash
docker compose -f docker-compose.prod.yml down      # 停止（保留数据）
docker compose -f docker-compose.prod.yml up -d      # 启动
docker compose -f docker-compose.prod.yml down -v    # ⚠️ 停止并删除数据（危险）
```

## 故障排查

| 症状 | 排查 |
|------|------|
| `curl` 连接超时 | DNS 未生效 / 安全组未放行 443 |
| 502 Bad Gateway | app 容器未启动或不在 `honlnk-gateway-net` 网络 |
| 502 + app 日志有 ECONNREFUSED | MySQL 未就绪，等一会或检查 mysql 容器健康状态 |
| 证书错误 | `sudo certbot certificates` 检查证书状态 |
| 登录失败 "管理员账号未初始化" | 未跑 seed，执行第 6 步 |
| MCP 调用 401 | Key 不存在或已禁用；在后台检查 |
